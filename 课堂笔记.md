### 字符串格式化（%占位符形式）

语法：

```python
"%s %d %f ... ..." % (变量, 变量, 变量, ....,...)
```

- `%s` 表示以字符串形式占位，等到数据的填充
- `%d` 表示以整数形式占位，等到数据的填充
- `%f`表示以浮点数形式占位，等到数据的填充
- %占位符的数量要和后续提供填充的数据的数量是一比一
- 任意数据类型可以用%s占位，但是不是任意类型都可以用%d或%f占位，要注意数据的转换关系
  - 如：%f占位，提供的数据是"abc"肯定是不可以的



示例：

```python
name = "周杰轮"
print("你是：%s" % name)
# 结果是： 你是 周杰轮

age = 11
height = 172.55
print("你是%s，身高：%f，年龄：%d" % (name, height, age))
# 结果：你是周杰轮，身高：172.550000，年龄：11
```

> 默认的%f占位，浮点数会使用6位小数精度，没有的话会补充0

### 字符串格式化的数字精度

基础写法是：

```python
%md
%m.nf
```

- `m` 表示数字的宽度
- `.n` 表示小数点的宽度
  - 仅对浮点数有效

示例：

```python
age = 11
print("%5d" % age)		# 结果：   11   11前面还有3个空格补充到5的宽度
print(f"{age:5d}")		# 结果：   11   11前面还有3个空格补充到5的宽度

height = 172.55
print("%7.1f" % height) 	# 结果：  172.6    前面有2个空格 小数点精确1位（有四舍五入效果）
print(f"{height:7.2f}")		# 结果：  172.6    前面有2个空格 小数点精确1位（有四舍五入效果）
```

### if语句

功能：做逻辑判断，即xxx是否满足，满足则做特定的事情。

语法：

```python
if 条件:
    条件满足后的代码
    ...
    ...
```

- `if` 是关键字，表示如果

- `:`不要忘记

> 当满足条件，则输出成年

### if else语句

功能：做逻辑判断，即满足xxx，则做特定的事情，否则，则做特定的事情

语法：

```python
if 条件:
    条件满足后的代码
    ...
else:
    条件满足后的代码
    ...
```

- else和if是同层次，即缩进是一致的
- else后面不需要写条件了（使用if的条件）
- `:`不要忘记
- else内的代码，不要忘记缩进4个空格



示例：

```python
money = int(input("请告诉我你还有多少钱："))

if money >= 10000:
    print("去买个新电脑")

else:
    print("洗洗睡吧")

```

## 循环

### while循环基础语法

概念：循环是对特定的操作，当条件满足则持续执行下去，直到条件不满足为止。

<img src="https://image-set.oss-cn-zhangjiakou.aliyuncs.com/img-out/2025/07/17/20250717182958.png" alt="image-20250717182958065" style="zoom:67%;" />

语法：

```python
while 条件:
    循环代码
    ...

```

- `while` 是关键字不要写错
- `条件` 必须产出布尔结果
- `缩进` 不要忘记



示例：

```python
while 1 == 1:
    print("你好")
```

> 这个代码满足循环语法，但是：无限循环



为了避免无限循环，我们自行在语法外引入限制，即循环三要素：

- 循环控制因子的创建
- 基于循环控制因子的条件判断
- 循环控制因子的更新

三要素合力，确保循环受限。



示例：

```python
# 打印我喜欢你 10次
# 基础循环，缺少控制，所以会导致无限执行
# while True:
#     print("我喜欢你")

# 控制循环在有限次数内执行，有3个要素
# 1. 循环控制因子（某个变量）
# 2. 循环控制条件（基于因子做判断）
# 3. 循环因子更新（修改因子的值）

# 控制因子
num = 0

# 控制条件（基于因子）
while num < 10:
    print("第%d次表白：我喜欢你" % num)
    num += 1        # 因子更新

```



### while循环嵌套

语法：

```python
while 条件:
    ...
    ...
    
    while 条件:
        ...
        ...
```

如上是一个2层嵌套

使用嵌套循环一定注意：

- `空格的缩进` ，因为Python是通过空格缩进决定层次关系
- 向右缩进，层次就低



案例：九九乘法表

```python
row = 1                 # 外层循环因子

while row <= 9:         # 基于因子的条件

    col = 1             # 内层循环因子
    while col <= row:   # 基于因子的条件
        print(f"{col}*{row}={col*row}", end="\t")
        col += 1        # 内层循环因子更新

    print()             # 外层循环功能，换一行
    row += 1            # 外层循环因子更新
```

流程：从待处理数据集中，挨个取出内容，赋予临时变量，然后执行循环语句，循环往复。

和while循环的流程对比，如下是while：

#### 可迭代对象（类型）

可迭代对象表示的是：支持挨个取出元素的类型，有如下类型属于可迭代类型：

- `字符串` 
- 列表
- 元素
- 等

所以，字符串是属于可迭代类型则可以用于for循环如：

#### range语句

range语句可以形成一个数字序列。

序列：有序的集合，比如：`1, 2, 3, 4, 5, ...` 就是数字序列

range语句形成的数字序列是`可迭代类型`（其本质是一个迭代器对象，等同于list列表）



语法1：

`range(num)`：形成从`0` 开始，到`num` 结束（不包括num本身）的数字序列

- 如：`range(5)`结果是：0, 1, 2, 3, 4



语法2：

`range(num1, num2)` ：形成从num1开始，到num2结束（不含num2）的数字序列

- 如：`range(5, 10)` 结果是：5, 6, 7, 8, 9



语法3：

`range(num1, num2, step)` ：形成从num1开始，到num2结束（不含num2）的数字序列，步长以step决定

- 如：`range(5, 10, 2)`结果是：5, 7, 9    (注意步长是2)

```python
    print(age)

print(age)
```

此代码中，age变量创建是在for循环内部，所以age的生效范围，仅在for循环内，即

- 第五行代码，是不OK的，因为脱离了age的生效范围



变量的生效范围一般是：

- 如果写在`:` 内，比如`if`  `elif`  `else`  `while`  `for`  `with` 等
  - 则生效在`:`所属的范围内
- 如果写在文件中，则全文件生效

>
> 反之，缩进靠右访问缩进靠左是OK的



### continue关键字

作用：可以控制循环的执行，具体是：当循环代码遇到`continue`的时候，将`跳过本次`循环，直接`进入下一次`循环

示例：


- 上述代码的`print("bbb")`将永远无法执行，因为遇到continue循环直接进入`下一次` 



### break关键字

```python
    break
    print("bbb")
```

- 上述代码仅输出一次`aaa` 后就结束，因为遇到break后，整个循环就结束了

### 参数

参数分为：

- 实际参数：在调用函数的时候，`实际`传入的参数



特点：

- 支持任意个传入参数

- 多个传入参数用逗号分隔，如：

- 实参的传入类型不受限制，如下：

  ```python
  def add(x, y, z):
  ```

### 全局变量

概念：没有写在函数内部的是全局变量，作用范围在函数内和外都生效。

#### global关键字

作用：在函数内部声明使用的变量是全局变量

演示：

```python
num = 100       # 作用域在于整个代码
def func_a():
    print(f"a: {num}")

def func_b():
    num = 200       # 这个num是一个和全局num同名的局部变量
    print(f"b: {num}")  # 此时访问的num是局部变量num

func_a()
func_b()

print(f"外部：{num}")
```

函数的输出结果是：

```shell
a：100
b：200
外部：100
```

函数b对num的修改是失效的，因为这个num在函数b中是写了一个和全局num同名的局部变量。

解决这个问题使用global关键字即可，如下代码：

```python
"""
全局变量就是：不定义在函数内部的变量
"""
# 全局变量
num = 100       # 作用域在于整个代码


def func_a():
    print(f"a: {num}")


def func_b():
    global num      # 在此函数内部使用的num是全局变量
    num = 200       # 这个num就是访问全局变量，修改他的值
    print(f"b: {num}")  # 此时访问的num是全局变量num


func_a()
func_b()

print(f"外部：{num}")

```

看`13` 行，通过global关键字可以在函数内部声明，num为全局变量，对num的所有使用都是用全局num

## 函数高阶应用

### 返回多个返回值

语法：

```python
def func(...):
    ...
    return 值,值,...,值		# 返回多个值，逗号分隔即可

# 调用的时候用多个变量接收
变量,变量,...,变量 = func()
```

示例

```python
def func(...):
    return 1, 2

x, y = func()
# x接收1
# y接收2
```



#### 原理

`return 1, 2` 这种写法，本质上是返回了`元组` ，即返回的是：`(1, 2)`

所以我们用2个变量接收，如：`x, y = func()` 会自动将元组的第一个元素赋值给x，第二个元素赋值给y



即：

```python
def func(...):
    return 1, 2

x = func()
# x的值是：(1, 2)   是一个元组
```



### 函数的多种传参形式

#### 位置传参

> 按照形式参数写的时候的位置，在调用函数的时候，传入实际参数。
>
> 实际参数和形式参数是一一对应的。



示例：

```python
def func(x, y, z):
    ...
    
func(1, 2, 3)
```

此时：1传入给x、2传入给y、3传入给z

- 数量一致
- 顺序一致



#### 关键字传参

语法：`形式参数名=实际参数值`

示例

```python
def func(x, y, z):
    ...
    
func(x=1, y=2, z=3)
```

传入参数的时候，是指定了形式参数名称，这样的话，可以不按照形式参数的顺序进行传入。

比如：

```python
def func(x, y, z):
    ...
    
func(y=2, z=3, x=1)
```

这样写，同样是 x得到1、y得到2、z得到3



同时关键字传入参数和位置参数可以混用，如下

```python
def func(x, y, z):
    ...
    
func(1, 2, z=3)
```

关键字传参的写法，必须在位置传参的右侧。即如下写法是`错误的` 

```python
func(x=1, 2, z=3)
```

x=1写在了位置参数的左侧，不符合要求



### 缺省参数（默认值参数）

在编写函数的时候，可以给某个参数提供默认值，此参数即可实现

- 调用的时候，不传入值，则使用默认值
- 如果传入值，则使用传入值



语法：

```python
def func(x, y, z=10):
    ...
```

在调用的时候：

- 写的是：`func(1, 2)` 可以看到只给x和y传入值，没有提供z的值，则z使用默认值10
- 写的是：`func(1, 2, 3)` 则z使用提供的3



注意，`带有默认值的参数，必须在没有默认值参数的右侧。`

如下写法是错误的：

```python
def func(x, y=1, z):
    ...
```

`y`这个带有默认值的参数，在非默认值参数z的左侧，是不符合语法要求，错误。



### 不定长参数（元组接收）

概念：可以写一个参数，表示不定长（参数数量不确定），将传入的参数全部收集到一个元组内。

语法：

```python
def func(x, y, *z):
	...
```

- x和y是普通参数
- z是不定长（元组接受）的参数，z就是一个元组，收集传入的实际参数



示例

```python
def func(x, y, *z):
	...

func(1, 2, 3, 4, 5, 6, 7, 8, 9)
```

如上代码：

- 1和2是提供给普通参数x和y
- 后续的3, 4, 5, 6, 7, 8, 9 全部存入了参数z中
- 参数z是一个元组



### 不定长参数（字典接收）

概念：可以写一个参数，表示不定长（参数数量不确定），将传入的参数全部收集到一个`字典`内。

语法：

```python
def func(x, y, **z):
	...
```

- x和y是普通参数
- z是不定长（元组接受）的参数，z就是一个元组，收集传入的实际参数

> 用字典接收是2个`*`号



示例

```python
def func(x, y, **z):
	...

func(1, 2, id=1,name='a',addr='shanghai',gender='男')
```

如上：

- 1和2传给普通参数x和y
- 后续的id=1,name='a',addr='shanghai',gender='男'会被收集为字典，存入变量z（字典）中
- 即z内部的数据是4个键值对



#### 不定长元组接收和不定长字典接收混用

```python
def func(name, age, *args, **kwargs):
    print(f"我是{name}, 年龄{age}岁, 我的爱好有：", end="")
    for i in args:          # args是元组
        print(i, end=" ")

    print()
    print("我的其它信息：", end=" ")

    for key in kwargs:      # kwargs是字典
        print(f"{key}: {kwargs[key]}", end="、")

    print()


func("周杰轮", 11, "唱", "rap", "跳", "打篮球", addr="石岩", id=123, money=20000)
```

如上代码，可以实现传入参数的2种不定长方式的混合使用。

- "周杰轮", 11是提供给name和age（基于位置传参）
- `"唱", "rap", "跳", "打篮球"` 提供给不定长参数`*args` ，被收集到args这个元组内
- `addr="石岩", id=123, money=20000`  提供给不定长参数`**kwargs` ，被收集到kwargs这个字典内



### 函数作为参数传递

概念：一般使用函数，都是传入的数据，如数字、字符串、数据容器。这种传递我们称之为：数据的传递。

除此以外，函数也可以接收函数作为参数传递，这样就不是传递的数据了，而是传递的`计算逻辑（函数）` 



示例

```python
"""
写函数的时候：
- 形式参数，可以接收函数传入
- 实际参数，真的可以传入一个函数

这种写法称之为函数式编程，核心思想是：传入的是计算逻辑
"""
def func(compute):
    # 被计算数据固定为1和2，要计算的逻辑取决于传入的compute函数
    result = compute(1, 2)
    print(result)


def compute1(x, y):
    return x + y
def compute2(x, y):
    return x - y
def compute3(x, y):
    return x * y
def compute4(x, y):
    return x / y


# 传入的是compute函数，而非变量和数据
# 传的是函数也就是传的是计算的逻辑
func(compute1)
func(compute2)
func(compute3)
func(compute4)
```

如上代码，传递的`compute 1 2 3 4` 就是4个不同的函数，代表4种不同的计算逻辑。



> 函数是可以作为参数传递的。



### lambda匿名函数

语法：

```python
lambda 形参列表: 函数体(一行)
```

作用：用来快速实现一个匿名的函数



- 带有名字的函数是：`def 函数名(...) ...` 
  - 有名字可以重复调用
- 匿名函数就是：`lambda x, y: ...` 
  - 没有名字，只是临时用一次

示例

```python
def func(compute):
    # 被计算数据固定为1和2，要计算的逻辑取决于传入的compute函数
    result = compute(1, 2)
    print(result)


# 调用func4次，完成+-*/
# 这些函数仅用一次，不用于重复使用，所以用lambda快速实现
func(lambda x, y: x + y)
func(lambda x, y: x - y)
func(lambda x, y: x * y)
func(lambda x, y: x / y)
```

- 上面调用的4次func，传入的4个不同的计算逻辑
- 这4个不同的计算逻辑就是4个匿名函数，只用一次



> 在以后的功能开发中，会遇到一些仅需要传入计算逻辑（函数）的函数，就可以用lambda快速实现。



匿名函数也可以有名字，用变量接收即可，如下：

```python
chengfa = lambda x, y: x * y
print(chengfa(1, 3))
```

这种写法一般不用，要是想给函数个名字，为何不用`def` 



## 文件操作

### 文件编码

> 计算机只认识0和1，包括内存、硬盘、网线等等这些IT设备，都是只认识0和1二进制。

为了能够存储丰富的文本数据在硬盘里面，我们需要将文本转换为二进制存储。



动作：

- 写入：文本 ->转换->二进制->存入硬盘
- 读取：读取硬盘->二进制->转换->文本

转换的动作就是二进制和文本之间的对照关系，这个对照关系，我们称之为编码。

![image-20250723175654500](https://image-set.oss-cn-zhangjiakou.aliyuncs.com/img-out/2025/07/23/20250723175655.png)



> 如上图，如果用编码表A存储数据，以编码表B解析数据，则会出现问题，无法解析或解析错误。
>
> 所以我们的文件在保持的时候有编码，在用的时候也有编码，要确保是一致的，否则有问题



当前，全球范围内最火的编码是：`UTF-8` ，这是全球通用编码表，支持全世界范围内绝大多数语言。

没特殊需求，首选UTF8，同时UTF8也是绝大多数软件的默认编码。



偶尔我们可能还会见到GBK编码，这个是中文专用编码，在某些老旧软件系统中还有。

## 异常

### 概念

异常称之为：`BUG（虫子）`   

异常：程序在运行的时候出现问题，导致程序无法继续的软件缺陷。





### 异常捕获

#### 为何

我们无法确保全部代码都是100%OK的

在

- 可能出现问题的代码上
- 做如果出现问题后的预案

这就是异常捕获的意义。



#### 语法

```python
try:
    可能有异常的代码
except 异常类型 as 变量:
    出现异常的处理代码
except 异常类型 as 变量:
    出现异常的处理代码
......
except 异常类型 as 变量:
    出现异常的处理代码
else:
    没有异常的处理代码
finally:
    有没有异常都会执行的代码
```

- `try` 只有在try内部的代码，才会被捕获异常
- `except` 是匹配机制，用来匹配特定异常
  - 特殊异常类型：`Exception` 所有异常的父类（顶级异常），任何异常都可以用`Exception` 抓住
- `else（可选）` ，没有异常的时候，会进入else的语句段
- `finally（可选）` 有没有异常，都会执行的

## 模块&包

### 模块

#### 概念

模块：一个完整的`.py`文件，里面包含相关Python代码（含有函数、类、变量等），称之为一个模块。

简单来说：一个python代码文件就是一个模块。



#### 用途

可以通过`import`  `from` 语句，从其它模块（代码文件）中导入相关的函数和变量和类等资源去使用



> 我们写代码不可能将全部代码都写到一个文件内，可以拆分成多个文件，通过import的方式从其它文件引用功能



#### 语法

```python
[from 包|模块] import (包|模块|函数|变量|类...) [as 别名]
```

- `[]`代表可选
- `|` 在语法中表示 或
- `()` 在语法中表示必写



示例

代码文件`B.py` 中有函数：`B1`  `B2`  `B3` 

在代码文件A中想要使用这三个函数，可以有如下写法：

方法1

```python
import B
B.B1()
B.B2()

# 或者带有别名
import B as b
b.B1()
b.B2()
```

方法2

```python
from B import B1
B1()
# 此时无法使用B2 B3 因为没有导入

# 或者
from B import B1, B2  # 此时无法使用B3 因为没有导入
B1()
B2()
```



方法3

```python
from B import *
B1()
B2()
B3()
```

> 这种写法不建议，因为不能确定B1 B2 B3和模块B的关系





### Python内置变量

#### `__name__` 变量

`__name__` 变量是Python自带的，任何一个Python代码文件内部，都是隐式（你不写也有）的附带这个变量。

功能：

- 当这个代码文件，作为Python程序执行的时候，此变量记录值是：`__main__`
- 当这个代码文件，作为模块被其它代码文件导入的时候，此变量记录值是：`代码文件名本身`

基于这个特性，我们可以区分这个文件是执行了还是被导入了。



常见用法：

```python
if __name__ == '__main__':
    xxx()
```

当此文件被作为程序执行，才会执行xxx函数

如果是作为模块被其它文件导入，上述的`if` 判断无法成立，则xxx不会执行



#### `__all__` 变量

此变量也是隐式存在于每一个代码文件中，默认值是 `*` 

它表示的是，如果文件被`from 模块 import *` 被导入，则内部的哪些函数可以被`*` 导入

默认，是全部

如果不想要全部的话，可以手动赋值此变量。示例如下：



文件B

```python
__all__ = ["B1", "B2"]

def B1():
    ...
def B2():
    ...
def B3():
    ...
```

当在文件A中使用语句：`from B import *` 则只会自动导入B的`B1` 和`B2` 2个函数，因为`__all__` 变量内记录的只有这2个

如果删除`__all__` 变量，则可以自动导入全部的函数。



### 包

概念：Python的包是一个特殊文件夹，这个文件夹内包含：

1. `__init__.py`的文件
2. 其它模块（.py）文件

简单来说，包就是管理非常多模块的一个层级。



比如新建一个包，只需要：

1. 创建一个文件夹，在文件夹内创建`__init__.py`文件
2. 将其它模块（`.py`）文件放入此文件夹

此时就得到了一个包。



示例：

文件夹`my_package`内有

- `__init__.py`文件
- 有`B.py` 模块文件，这个模块内提供`B1`  `B2`   `B3` 三个函数



如果想要使用这三个函数可以：

```python
# 方式1（不推荐不好写）
import my_package.B
my_package.B.B1()
my_package.B.B2()
my_package.B.B3()

# 方式2 推荐好写
from my_package import B
B.B1()
B.B2()
B.B3()

# 方式3 也推荐
from my_package.B import B1
B1()  	# 此时无法用B2 和B3  因为没导入

# 方式4 不推荐 可读性不好
from my_package import *
B.B1()
B.B2()
B.B3()

# 方式5 不推荐 可读性不好
from my_package.B import *
B1()
B2()
B3()
```



`__init__.py`文件内置变量`__all__`

在这个文件内同样有`__all__` 变量，可以控制`from 包 import *` 语句中，哪些模块被自动导入。

示例：

包`my_package` 内有 3个模块`A.py`  `B.py`  `C.py` 以及`__init__.py`文件

在`__init__.py` 文件内是空的，则

- `from my_package import *`   这个写法，无法使用A、B、C任何的模块

在`__init__.py` 文件内写上：`__all__ = ["A", "B"]` 则

- `from my_package import *` 这个写法，可以使用A和B2个模块，无法用C这个模块

- `ls -l /usr/bin | grep gst | wc -l`

  - 将`ls -l /usr/bin | grep gst`的结果作为wc -l的输入

### 运行模型

无论什么操作系统，都可以在终端中运行：

```shell
ollama run deepseek-r1:1.5b
```

即可运行`1.5b`参数的deepseek-r1蒸馏模型。

首次运行需要联网下载，大概1.1GB空间。



### 蒸馏模型

我们运行的模式都是蒸馏模型，即非完整模型。

蒸馏：按照需求进行模型提纯（保留核心网络并进行简化）



我们选择的`1.5b`参数就是蒸馏后的（提纯和简化），为了方便低性能设备的运行。

完整的DeepSeek模型，简单来说显卡不花个`20W`基本没啥体验。



> 模型运行是基于GPU（显卡），因为GPU特点：核多（并行计算能力超强）
>
> - CPU：核心少，比如`i7-13650HX` 仅仅有20个核心，这些核心每一个都超强，CPU适合通用计算
> - GPU（显卡）：核心超多，动辄上千的流处理器（核心），每个核心都垃圾（对比CPU），GPU适合做并行计算
>
> 因为模型的神经网络计算，一般来说是量大，而非计算难度高。
>
> 量大的计算，用显卡超爽（核多）

### session_state

Streamlit工具编写的网页会定期刷新，等同于每隔一段时间，重新执行整个Python代码。

比如：

```python
import streamlit as st

st.....
st.....

d = {}

while True:
    d[key] = xxx	# 每次循环的Key 假设永不重复
```

我们代码写的无限循环，不停的给字典添加信息，但是这个字典能够记录的信息永远只有1个key

因为这段代码被Streamlit执行起来，相当于每隔一段时间都定期执行。

则每一次执行字典`d` 都会被赋值为空字典。原有记录信息会丢失。



如果想要在网页运行的过程中存储下来信息，可以用Streamlit内置的字典，这个字典将不会丢失内容。

即将代码改为：

```python
import streamlit as st

st.....
st.....


while True:
    st.session_state[key] = xxx	# 每次循环的Key 假设永不重复
```

`st.session_state` 就是一个字典，我们用这个字典，记录的内容就不会丢失。

```python
import streamlit as st
import langchain_util as utils

# 添加标题
st.title("黑马聊天机器人（阿里云通义千问模型版）")

st.divider()

if "message" not in st.session_state:
    st.session_state["message"] = [{"role": "assistant", "content": "你好我是人工智能机器人，有什么可以帮到您!"}]
    # st.session_state["message"].append({"role": "assistant", "content": "你好我是人工智能机器人，有什么可以帮到您!"})

# list.append({"role": "user", "content": "内容"})
# state["message"]  =   [ {"role": "user", "content": "内容"} , {"role": "user", "content": "内容"}]
# 每一个消息的对话分为2部分： 角色，内容
# 机器人  角色：assistant   内容：说的话
# 人     角色：user         内容：说的话
for message in st.session_state["message"]:
    st.chat_message(message["role"]).write(message["content"])

# 用户输入 在页面下方添加用户输入栏
prompt = st.chat_input()

if prompt:          # 如果prompt有内容 表示用户提问

    # 首先将用户的提问在页面输出
    st.chat_message("user").write(prompt)

    # 把用户提问这个对话保存到session_state内
    st.session_state["message"].append({"role": "user", "content": prompt})

    # 调用AI回答
    with st.spinner("正在思考中..."):        # 转圈圈的加载框
        res = utils.get_response(prompt, "sk-fa5b080ac78b4323b13f733b50f1d5c0")

    # 将AI回答信息写到屏幕上
    st.chat_message("assistant").write(res)

    # 将AI回答的消息记录到session_state内
    st.session_state["message"].append({"role": "assistant", "content": res})
```
